How It Works

From docs to decision
in one run

RAGBench evaluates the layer that matters most — retrieval — using industry-standard IR metrics.

1
Ingest & Chunk
Your documents are loaded, split into overlapping word-level chunks, and indexed. Configurable chunk size and overlap.
2
Build Retrieval Indexes
Three indexes are built simultaneously — BM25 lexical index, TF-IDF vector, and FAISS dense embedding index (sentence-transformers all-MiniLM-L6-v2).
3
Run Benchmark Queries
22 ground-truth queries run against each retriever. Each query has a gold-standard relevant chunk set for evaluation.
4
Compute IR Metrics
Recall@5, MRR@10, and nDCG@10 computed per retriever. Results stored in SQLite for trend tracking.
5
Verdict & Action
Dashboard declares a winner, explains why, and flags any regressions. Failure analysis shows exactly which queries need improvement.
